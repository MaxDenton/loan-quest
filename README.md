# Loan Quest 💰

Одностраничное приложение (SPA) для подачи заявки на займ с валидацией данных и финальным подтверждением.

**Архитектура:** Feature-Sliced Design (FSD)  
**Сборщик:** Vite

## 🚀 Технологии

- **React 19** — UI библиотека
- **TypeScript** — типизация
- **React Router v7** — навигация между формами
- **TanStack Query** — управление API запросами и кэширование
- **shadcn/ui** — UI компоненты
- **Tailwind CSS** — стилизация
- **react-imask** — маска для ввода телефона
- **Vite** — современный сборщик

## ✨ Возможности

- ✅ Три последовательные формы с валидацией
- ✅ Маска ввода телефона (0XXX XXX XXX)
- ✅ Загрузка списка мест работы из API (dummyjson.com)
- ✅ Кэширование данных API через TanStack Query
- ✅ Сохранение данных формы в sessionStorage
- ✅ Защита от прямого доступа к URL шагов
- ✅ Анимации переходов между шагами
- ✅ Модальное окно подтверждения
- ✅ Финальный экран "Fine!"

## 🛠 Установка

### Требования
- Node.js >= 16.x
- npm / yarn / pnpm

### Шаги установки

```bash
# Клонируйте репозиторий
git clone https://github.com/MaxDenton/loan-quest.git
cd loan-quest

# Установите зависимости
npm install

# Запустите проект (режим разработки)
npm run dev

# Соберите production версию
npm run build
```

Приложение откроется в браузере по адресу: **http://localhost:3000**

## 🧪 Как проверить работоспособность

### 1. Запуск приложения
```bash
npm start
```

### 2. Пройдите все шаги формы

#### Шаг 1: Личные данные
- **Телефон**: введите любой номер в формате 0XXX XXX XXX (например, `9999123456`)
- **Имя**: введите имя (например, `Иван`)
- **Фамилия**: введите фамилию (например, `Петров`)
- **Пол**: выберите из списка
- Нажмите **"Далее"**

#### Шаг 2: Адрес и место работы
- **Место работы**: выберите из списка (данные загружаются из API)
- **Адрес проживания**: введите адрес (например, `ул. Ленина, 5`)
- Нажмите **"Далее"**

#### Шаг 3: Параметры займа
- **Сумма займа**: выберите ползунком ($200–$1000)
- **Срок займа**: выберите ползунком (10–30 дней)
- Нажмите **"Подать заявку"**

#### Шаг 4: Подтверждение
- Дождитесь появления модального окна
- Проверьте текст: *"Поздравляем, [Фамилия] [Имя]. Вам одобрена [сумма] на [срок] дней."*
- Нажмите **"Закрыть"**

#### Шаг 5: Финальный экран
- Должен отобразиться экран с надписью **"Fine!"**
- Нажмите **"Начать новую заявку"** для сброса данных

### 3. Production сборка

```bash
# Создайте production сборку
npm run build

# Проверьте локально (опционально)
npx serve -s build
```

Сборка будет доступна в папке `build/`

## 🔒 Особенности реализации

### Защита роутов (ProtectedRoute)
- Прямой доступ к `/step/2`, `/step/3`, `/confirmation`, `/done` защищён
- Автоматический редирект на последний заполненный шаг
- Валидация данных перед переходом

### Сохранение данных (sessionStorage)
- Данные формы сохраняются в `sessionStorage`
- Данные сохраняются при обновлении страницы
- Данные очищаются после успешной отправки

### API (TanStack Query)
- Workplace данные загружаются из `https://dummyjson.com/products/category-list`
- Отправка заявки через `https://dummyjson.com/products/add`
- При недоступности API используются fallback данные
- Кэширование через TanStack Query (10 минут)
